# Prompt 05 - Public Pages and Site Experience

Gunakan prompt ini untuk mengerjakan **Stage 5** dari proyek portfolio website. Fokus pada halaman publik dan pengalaman browsing utama.

## Prompt

Bangun halaman publik untuk portfolio website ini dengan metode **Multi-Stage Prompting**. Fokus hanya pada halaman publik, layout, dan alur pengalaman pengguna.

## Project Context

- Website bersifat `multi-page`
- Public pages:
  - `/`
  - `/about`
  - `/skills`
  - `/projects`
  - `/contact`
- Gaya visual: modern clean
- Projects ditampilkan sebagai card grid
- Konten sebaiknya berasal dari data dinamis bila memungkinkan

## Stage Goal

Membuat pengalaman halaman publik yang rapi, profesional, dan konsisten di seluruh website.

## Multi-Stage Instructions

### Step 1 - Review Information Architecture

- Tinjau struktur halaman publik.
- Tentukan peran utama tiap halaman.

### Step 2 - Build Shared Layout

- Buat navbar, footer, container system, dan page layout yang konsisten.
- Pastikan navigasi antar halaman jelas dan enak digunakan.

### Step 3 - Build Public Pages

- Bangun:
  - Home
  - About
  - Skills
  - Projects
  - Contact

### Step 4 - Shape the Content Experience

- Home harus menjadi landing page yang kuat namun tidak terlalu padat.
- Projects page harus menampilkan grid yang clean dan mudah dipindai.
- Skills page harus jelas dan mudah dibaca.
- About page harus terasa personal namun tetap profesional.

### Step 5 - Verify

- Pastikan semua halaman publik terhubung dengan baik.
- Pastikan pengalaman visual dan struktur kontennya konsisten.
- Ringkas apa yang dibangun pada stage ini.

## Expected Output

- Public website lengkap
- Layout konsisten antar halaman
- UX dasar yang kuat untuk pengunjung

## Constraints

- Jangan fokus pada dashboard admin di stage ini
- Jaga visual tetap clean, modern, dan tidak generik
- Gunakan animasi hanya jika mendukung pengalaman pengguna
