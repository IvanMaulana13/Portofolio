# Prompt 04 - Admin Dashboard and Content Management

Gunakan prompt ini untuk mengerjakan **Stage 4** dari proyek portfolio website. Fokus pada fondasi dashboard admin dan modul pengelolaan konten utama.

## Prompt

Lanjutkan pembangunan portfolio website menggunakan pendekatan **Multi-Stage Prompting**. Pada stage ini, bangun dashboard admin yang sederhana, rapi, dan efektif untuk mengelola konten website.

## Project Context

- Website portfolio modern clean
- Single admin login sudah tersedia atau akan digunakan
- Konten utama harus dapat diubah tanpa edit kode
- Area admin mencakup:
  - overview
  - projects
  - skills
  - messages
  - profile

## Stage Goal

Membangun dashboard admin yang memungkinkan pemilik website mengelola konten portfolio secara mandiri.

## Multi-Stage Instructions

### Step 1 - Review Current State

- Tinjau struktur admin yang sudah ada.
- Tentukan layout admin yang sederhana dan scalable.

### Step 2 - Build Admin Foundation

- Buat layout dashboard admin.
- Tambahkan navigasi admin yang jelas.
- Buat halaman overview sebagai entry point.

### Step 3 - Build Content Modules

- Implementasikan pengelolaan data berikut:
  - Projects
  - Skills
  - Profile
  - Social Links
  - Messages read/view management

### Step 4 - Define CRUD UX

- Pastikan flow tambah, edit, hapus, dan lihat data terasa sederhana.
- Gunakan form dan tabel atau list yang rapi.
- Fokus pada usability, bukan tampilan dashboard yang berlebihan.

### Step 5 - Verify

- Pastikan admin dapat mengelola konten inti tanpa menyentuh source code.
- Ringkas modul yang dibuat dan kemampuan dashboard saat ini.

## Expected Output

- Dashboard admin usable
- Modul CRUD utama siap dipakai
- Pengelolaan konten portfolio menjadi dinamis

## Constraints

- Jaga tampilan admin tetap clean dan ringan
- Hindari fitur dashboard yang tidak diperlukan
- Fokus pada kebutuhan nyata pemilik portfolio
