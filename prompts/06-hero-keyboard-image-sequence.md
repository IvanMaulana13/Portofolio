# Prompt 06 - Hero Keyboard Image Sequence

Gunakan prompt ini untuk mengerjakan **Stage 6** dari proyek portfolio website. Fokus hanya pada hero visual berbasis image sequence keyboard.

## Prompt

Implementasikan hero animation untuk portfolio website ini menggunakan pendekatan **Multi-Stage Prompting**. Hero visual utama berupa keyboard image sequence yang tampil di samping teks hero.

## Project Context

- Website portfolio modern clean
- Hero section terdiri dari teks utama dan visual keyboard
- Aset image sequence keyboard sudah tersedia
- Animasi dipakai sebagai pembeda visual utama website
- Hero visual tidak berfungsi sebagai navigasi interaktif

## Stage Goal

Membuat hero section yang terasa unik, premium, modern, dan tetap ringan secara performa.

## Multi-Stage Instructions

### Step 1 - Inspect Assets and Context

- Tinjau aset image sequence yang tersedia.
- Identifikasi ukuran, jumlah frame, dan kebutuhan performa.

### Step 2 - Choose Rendering Strategy

- Tentukan pendekatan implementasi yang paling efisien.
- Prioritaskan solusi yang smooth namun tetap ramah performa, terutama di mobile.
- Jelaskan secara singkat alasan pendekatan yang dipilih.

### Step 3 - Build the Hero Experience

- Tempatkan keyboard image sequence di samping teks hero.
- Pastikan komposisi visual tetap clean dan seimbang.
- Tambahkan motion secukupnya agar hero terasa hidup, tetapi tidak berlebihan.

### Step 4 - Handle Performance and Fallback

- Optimalkan loading image sequence.
- Tambahkan fallback jika diperlukan untuk device atau kondisi yang lebih lemah.
- Pastikan mobile experience tetap baik.

### Step 5 - Verify

- Pastikan hero tampil baik di desktop, tablet, dan mobile.
- Pastikan sequence tidak mengganggu readability konten utama.
- Ringkas strategi implementasi dan optimasinya.

## Expected Output

- Hero section dengan keyboard image sequence yang bekerja baik
- Performa tetap terjaga
- Visual hero terasa khas dan modern

## Constraints

- Jangan ubah hero menjadi navigasi interaktif
- Prioritaskan pengalaman visual yang elegan, bukan ramai
- Hindari implementasi yang membebani initial load secara berlebihan
